Detailed Description

Manages the loading of a OgreMax .scene file. Use a CustomOgreMaxSceneCallback to set up your scene: CustomOgreMaxScene scene; CustomOgreMaxSceneCallback * callback = new MyCallback(someogrepointers); scene.load(file, OPTIONS, callback);

Skips the externals in the file

SKIP_TERRAIN 

Skips the terrain in the file

SKIP_QUERY_FLAG_ALIASES 

Skips scene level query flags in the file

SKIP_VISIBILITY_FLAG_ALIASES 

Skips scene level visibility flags in the file

SKIP_RESOURCE_LOCATIONS 

Skips scene level resource locations in the file

NO_ANIMATION_STATES 

Indicates animation states should not be created for node animation tracks

NO_EXTERNALS 

Indicates scene 'externals' should not be stored. They will, however, still be loaded and the CreatedExternal() scene callback will be called

NO_FILE_SYSTEM_CHECK 

By default, the OgreMaxScene::Load() checks to see if the file that was passed in exists in the file system (outside of the configured resource locations). This flag disables that logic

SET_DEFAULT_LIGHTING 

Indicates that a light should be created if the loaded scene doesn't contain a light Any lights created as a result of setting the default lighting are not passed to the scene callback

FILE_NAME_CONTAINS_CONTENT 

Indicates that the 'fileNameOrContent' parameter in Load() should be treated as the scene XML. In other words, no file is loaded from Ogre's file system.
